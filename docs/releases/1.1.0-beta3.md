# Release 1.1.0-beta.3


##  New Features

- Style inheritance on add: New columns automatically inherit the style (background, border, text color, separator, etc.) of their neighbor column. Toggleable in Settings > General > "Inherit style on add" (default: on).
- Ctrl+Click + for opposite column type: Hold Ctrl (or Cmd on macOS) and click the + button to insert the opposite type — stacked columns get a non-stacked sibling, non-stacked columns create a new stacked group with the neighbor.
- Third-party plugin autocomplete support: Column editors now bridge Obsidian's EditorSuggest API, enabling suggestions from third-party plugins (e.g. Iconize : trigger) to work inside columns.
- Stack group splitting: Selecting a contiguous subset of stacked columns and toggling "Stacked" off now splits them into a separate stacked group instead of incorrectly nesting them.

## Improvements

- Default secondary background: All built-in templates (2/3/4-column, nested, sidebar, stacked+wide, Cornell notes) now use b:secondary as the default column background for better visual distinction.
- Centered drag handle: The column drag handle is now a horizontal three-dot indicator (···) centered at the top of each column. On hover, it highlights with the accent color.
- Context-aware + tooltip: The add column button tooltip now reflects the column context — "Add column to the right" for non-stacked, "Add stacked item below" for stacked columns.
- Link color inheritance: Custom text color applied via the context menu now correctly applies to links (internal, external, tags) within the column.
- Non-intrusive column header: The column header overlay no longer occupies vertical space when hidden — it only appears on hover/edit without affecting layout.

## Bug Fixes

- Ctrl+click conflict resolved: Ctrl+clicking the + button no longer triggers column multi-select; it correctly performs the opposite-type add action.

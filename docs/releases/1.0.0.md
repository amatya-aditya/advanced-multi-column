# Advanced Multi Column v1.0.0

Initial release.

## Highlights

- **Marker-based syntax** — simple Obsidian comments (`%% col-start %%`, `%% col-break %%`, `%% col-end %%`) define columns. Notes stay plain markdown.
- **Live Preview + Reading View** — columns render in both editing and reading modes. Each can be toggled independently.
- **Nested columns** — place column blocks inside other columns, unlimited depth.
- **Drag to reorder** — grab the column handle (or Alt+drag the body) to rearrange columns, even across nested containers.
- **Resize by dragging** — drag the vertical divider between columns to adjust widths.

## Full Feature List

### Syntax & Layout
- `%% col-start %%` / `%% col-end %%` delimit a column block.
- `%% col-break %%` starts each column. Content before the first break is ignored.
- Width control: `%% col-break:40 %%` or `%% col-break:w:40 %%` sets a column to 40%.
- Nested blocks: place a full `col-start` … `col-end` region inside any column.
- 5 insert commands: 2-wide, 3-wide, 4-wide, custom count, nested template.

### Editing
- Click any column preview to enter inline edit mode.
- Tab / Shift+Tab to cycle through columns.
- Esc commits changes and exits edit mode.
- `[[` triggers wikilink autocomplete from vault files.
- Ctrl/Cmd+B for bold, Ctrl/Cmd+I for italic.
- Paste an image to auto-save and insert `![[image.png]]`.

### Styling
- Right-click any column to open the style popover.
- Per-column styling: background, border color, text color, border toggle, horizontal dividers.
- Container-level styling: same options applied to the whole block.
- Style tokens in markers for portable, file-level styles:
  - `b:` — background (12 options: transparent, primary, secondary, alt, accent-soft, red-soft, orange-soft, yellow-soft, green-soft, cyan-soft, blue-soft, pink-soft)
  - `bc:` — border color (11 options: gray, accent, muted, text, red, orange, yellow, green, cyan, blue, pink)
  - `t:` / `tc:` — text color (same 11 options)
  - `sb:` — show border (1/0, true/false, yes/no, on/off)
  - `h:` / `hd:` — horizontal dividers (same boolean values)
- Global style settings in the settings tab (background, borders, corner radius, divider width/style/color).
- Style target: apply global settings to all columns or a specific column index.

### Settings
- General: enable/disable live preview and reading view, default column count, minimum column width, drag handles toggle.
- Appearance: style target, container styling, vertical and horizontal divider configuration.
- About: version info, support links, community link.

### Compatibility
- Legacy callout-based column CSS (`[!col]` / `[!col-md-*]`) still shipped for backward compatibility.
- Works on desktop and mobile (not desktop-only).
- Minimum Obsidian version: 0.15.0.

## Technical Notes
- Live Preview uses a CM6 StateField for decoration.
- Reading View uses a post-processor with CSS `:has()` driven content switching (no MutationObserver feedback loops).
- Styles are applied via `adoptedStyleSheets` for runtime CSS injection.
